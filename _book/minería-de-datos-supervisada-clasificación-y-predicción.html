2 Minería de Datos Supervisada: Clasificación y Predicción

2.1 Regresión

\[y=f(x_1,x_2, \ldots)\]

  • \(y\) Variable de resultado, dependiente, solo tenemos a una \(y\).
  • \(x_1, x_2, \ldots\), variables de control, independientes.

A partir de estas variables:

  • ¿Cuál es la relación de \(x\) sobre \(y\)?
    • Lineal

\[y_i=\beta_0+\beta_1 x_{i1}+\beta_2 x_{i2}+\ldots+\epsilon_i\] \[E[y_i]=E[\hat{\beta}_0+\hat{\beta}_1 x_{i1}+\hat{\beta}_2 x_{i2}+\ldots]\]

\[\frac{dy}{d x_1}= \beta_1\]

Nota: Diferenciar que la regresión busca establecer relaciones basadas en los datos y no asi un proceso causal.

  • Polinomial

  • Etc; No lineal,

  • Conocer la naturaleza de \(y\) y las variables \(x\)

    • \(Y\) es cuanti (real), \(X\) mixtas: Modelos lineales (MCO). Busca \(E[y|x]\).
    • \(Y\) es cuanti (discreta \(\ge 0\)), \(X\) cuanti: Regresión de Poisson o Binomial Negativa.
    • \(Y\) es cuali nominal binaria, \(X\) mixtas: LOGIT / PROBIT.
    • \(Y\) es cuali ordinal, \(X\) mixtas: Logit / Probit ordenados.
    • \(Y\) con heterocedasticidad o interés en extremos: Regresión Cuantílica. Busca \(Q_\tau(y|x)\).
    • \(Y\) censurada (ej. ingresos truncados en 0): Modelo TOBIT.

2.1.1 Pasos para la implementación de una regresión

  1. Pregunta de investigación, revisión de literatura
  2. Base datos lista para el modelo (Unidad de investigación)
  3. Establecer la relación interés
  4. Definir el modelo de interés
  5. Optimizar el modelo
  6. Validar el modelo
  7. Analizar/Predecir a partir del modelo

2.1.2 Regresión lineal

\[y \in IR\]

\[x_1, x_2, \ldots (mixtas)\]

\[y=\beta_0+\beta_1 x_{1}+\beta_2 x_{2}+\ldots+\epsilon_i\]

\[\epsilon_i\sim N(0, \sigma^2)\]

Las variables \(x\) son independientes mutuamente.

Causalidad vs Correlación

2.1.3 Probit y Logit

Estrategia, llevar valores binarios a valores continuos. Mediante una función de enlace (\(F(Y)\)).

\[F(Y)=Y'=X \beta +\epsilon\]

Probit:

\[Y=\Phi (X \beta +\epsilon)\] \[\phi^{-1}(Y)=X \beta +\epsilon\]

\[Y'=X \beta +\epsilon\]

El enlace \(F(Y)=\Phi^{-1}(Y)\), es conocida como probit.

Logit:

\[logit(Y)=log\left(\frac{Y}{1-Y}\right)=X\beta+\epsilon\]

\[Y=\frac{e^{X\beta+\epsilon}}{1+e^{X\beta+\epsilon}}\]

2.1.4 Regresión Cuantílica

El modelo no se enfoca en la media condicional, sino en el cuantil \(\tau\):

\[Q_{y|x}(\tau) = \beta_0(\tau) + \beta_1(\tau) x_{1} + \beta_2(\tau) x_{2} + \ldots\]

  • Variable dependiente: \(y \in \mathbb{R}\)
  • Regresores: \(x_1, x_2, \ldots\) (pueden ser continuos o categóricos).
  • Función de pérdida (Check Function):

\[{\displaystyle \rho _{\tau }(u)=u(\tau -\mathbb {I} _{(u<0)})={\begin{cases}(\tau -1)u,&{\text{if }}u<0,\\\tau u,&{\text{if }}u\geq 0,\end{cases}}}\]

A diferencia de OLS/MCO, donde los errores \(\epsilon_i \sim N(0, \sigma^2)\), aquí se asume que el cuantil \(\tau\) de la distribución de los errores condicionales es cero: \(P(y_i \leq x_i' \beta_\tau | x_i) = \tau\).

2.1.5 Recomendación

  • Lo visto en este tema tiene una limitación, es el tratamiento sobre muestras autoponderadas es decir su alcance es limitado.
  • Para incorporar este tratamiento en encuestas por muestreo se recomienda usar la librería survey y srvyr
  • Se puede usar los métodos de agrupamiento para mejorar el rendimiento del modelo, muchas veces como una variable de estrato.
  • Para garantizar que el modelo este libre de multicolinealidad se recomienda usar componentes principales

2.2 Clasificación

La clasificación es una tarea de aprendizaje automático supervisado que consiste en identificar la categoría o “clase” a la que pertenece una nueva observación basándose en un modelo construido a partir de datos históricos ya etiquetados. A diferencia de la regresión, que predice valores numéricos continuos, la clasificación trabaja con una variable objetivo cualitativa denominada “clase”, la cual se divide en categorías llamadas “niveles”.

De manera formal, se busca entender la relación:

\[y = f(x)\]

Donde \(y\) es la variable dependiente (categórica) y \(x\) representa el conjunto de variables independientes o predictores. Esta técnica es fundamental en la analítica predictiva para predecir comportamientos futuros basándose en datos estructurados y no estructurados.

Ejemplos de aplicación:

  • Negocios: Identificación de clientes con riesgo de impago (loan default) o probabilidad de cancelar un servicio (churn).
  • Salud: Diagnóstico de patologías a partir de síntomas, temperatura o radiografías.
  • Seguridad: Filtrado de mensajes de spam y detección de actividad criminal o intrusiones en redes.

Actividad en el Aula (5 min): “Cazadores de Clases”

Identifiquen un problema de clasificación en su área (economía o negocios) y definan:

  • ¿Cuál es la variable objetivo (y)?.
  • ¿Es binaria o multinomial?.
  • ¿Qué predictores (x) usarían?.

2.2.1 Pasos Genéricos

Un proyecto de clasificación exitoso no depende solo del algoritmo, sino de un flujo de trabajo riguroso y repetible. Se identifican cinco pasos fundamentales Lantz (2013):

  1. Recolección de datos: Identificación de variables estructurales \(X\) y la variable objetivo \(Y\).
  2. Exploración y Preparación (IDA/EDA): Limpieza, manejo de valores perdidos y normalización (vital para algoritmos basados en distancia). Esta fase consume aproximadamente el 80% del esfuerzo del proyecto.
  3. Entrenamiento del modelo: Selección del algoritmo adecuado basado en la naturaleza del problema.
  4. Evaluación del desempeño: Uso de métricas para verificar si el algoritmo aprendió patrones o solo memorizó los datos.
  5. Mejora del modelo: Ajuste de parámetros o uso de métodos de ensamble para maximizar la precisión.

2.2.2 Validación y Muestreo: Estrategias contra el Sesgo

Para garantizar que el modelo sea capaz de generalizar ante datos no vistos, es imperativo realizar particiones del dataset.

2.2.2.1 A. El Método Holdout y el “Three-Way Split”

Tradicionalmente se divide el dataset en Entrenamiento (70-80%) y Prueba (20-30%). Sin embargo, para problemas complejos o con muchas variables categóricas, se recomienda una división triple Zumel and Mount (2019):

  • Entrenamiento: Para construir el modelo.
  • Calibración: Para ajustar umbrales de decisión (thresholds) y evitar el sobreajuste en el plan de tratamiento de datos.
  • Prueba (Test): Reservado estrictamente para la evaluación final.

2.2.2.2 B. Validación Cruzada (k-fold CV)

Divide los datos en \(k\) particiones aleatorias (folds). El modelo se entrena \(k\) veces, rotando el fold de validación en cada iteración. Este método es el estándar para estimar la estabilidad del modelo Lantz (2013).

2.2.2.3 C. Criterios Ad-hoc y Balanceo

  • Estratificación: Es vital asegurar que la proporción de las clases sea similar en cada partición, especialmente en eventos raros.
  • Base Error Rate: Un modelo solo es útil si su precisión supera la proporción de la clase mayoritaria (ej. si el 70% de los préstamos son buenos, cualquier modelo debe ser mejor que el 70% de exactitud)

2.2.2.4 D. Tratamiento del Desbalance de Clases (Class Imbalance)

El problema de desbalance ocurre cuando el dataset tiene una gran mayoría de registros pertenecientes a una sola categoría. Esto es común en casos de detección de fraude, donde el evento de interés (fraude) suele ser una pequeña fracción del total (ej. menos del 10%).

  • El Riesgo: Muchos algoritmos fallan al intentar minimizar el error global, sesgándose hacia la clase mayoritaria y fallando en identificar la minoritaria. En estos casos, la Exactitud deja de ser una métrica útil y se debe priorizar la Sensibilidad o el Kappa.
  • Técnicas de Re-muestreo (Resampling): Se utilizan para “facilitar el aprendizaje” del algoritmo ajustando las proporciones en el set de entrenamiento:
  • Sobre-muestreo (Oversampling): Incrementa artificialmente el número de ejemplos de la clase minoritaria (ej. duplicándolos o creando casos sintéticos) hasta alcanzar una proporción más equilibrada, como un 50/50.
  • Bajo-muestreo (Undersampling): Reduce el número de ejemplos de la clase mayoritaria para igualarlos a la minoritaria. Es útil en Big Data para reducir costos computacionales.
  • Criterio Fundamental: Aunque el entrenamiento se realice con datos balanceados (50/50) para mejorar la detección de patrones, la evaluación final en el set de prueba siempre debe hacerse con la distribución real de los datos para conocer el rendimiento real del modelo en producción.

2.2.3 Taxonomía de Métodos de Clasificación

Existen múltiples familias de algoritmos según su enfoque matemático. A continuación, se listan los principales métodos y sus variantes:

  • Regresión Lineal y Generalizada:
    • Regresión Logística (Logit): Binaria, Multinomial y Ordenada.
    • Probit: Basada en la distribución normal acumulada.
  • Métodos Probabilísticos:
    • Naive Bayes: Incluye variantes como Gaussiana, Multinomial y Bernoulli.
    • Redes Bayesianas.
  • Aprendizaje Basado en Instancias (Lazy Learning):
    • k-Nearest Neighbors (kNN): Clasificación por vecindad y distancia.
    • Learning Vector Quantization (LVQ).
  • Métodos de “Divide y Vencerás” (Árboles y Reglas):
    • Árboles de Decisión: CART (rpart), ID3, C4.5, C5.0, CHAID.
    • Reglas de Clasificación: 1R, RIPPER.
  • Métodos de “Caja Negra” (No Lineales):
    • Redes Neuronales Artificiales (ANN): Perceptrón, Multicapa con Backpropagation y Deep Learning.
    • Máquinas de Soporte Vectorial (SVM): Lineales y con funciones de Kernel.
  • Métodos de Ensamble (Avanzado):
    • Bagging y Random Forest.
    • Boosting: AdaBoost, Gradient Boosting (XGBoost).
    • Stacking y Blending.

2.2.4 Medidas de Calidad y Evaluación del Desempeño

La evaluación del desempeño permite determinar si el modelo es exitoso en su propósito y capaz de generalizar ante datos no vistos en lugar de simplemente memorizar el entrenamiento.

2.2.4.1 A. La Matriz de Confusión

Es una herramienta esencial que tabula las clasificaciones reales frente a las predicciones del modelo. Para un caso binario, se estructura de la siguiente forma:

Real: Negativo Real: Positivo
Predicho: Negativo Verdadero Negativo (TN) Falso Negativo (FN)
Predicho: Positivo Falso Positivo (FP) Verdadero Positivo (TP)
  • Verdadero Positivo (TP): Clasificado correctamente en la clase de interés.
  • Verdadero Negativo (TN): Clasificado correctamente fuera de la clase de interés.
  • Falso Positivo (FP): Clasificado incorrectamente en la clase de interés (Error Tipo I).
  • Falso Negativo (FN): Clasificado incorrectamente fuera de la clase de interés (Error Tipo II).

2.2.4.2 B. Métricas Derivadas

  • Exactitud (Accuracy): Proporción de predicciones correctas sobre el total. \[Accuracy = \frac{TP + TN}{TP + TN + FP + FN}\]
  • Tasa de Error: Proporción de casos mal clasificados (\(1 - Accuracy\)). \[Error Rate = \frac{FP + FN}{TP + TN + FP + FN}\]
  • Sensibilidad (Recall / TPR): Capacidad del modelo para identificar correctamente los casos positivos. \[Sensitivity = \frac{TP}{TP + FN}\]
  • Especificidad (TNR): Capacidad de identificar correctamente los casos negativos. \[Specificity = \frac{TN}{TN + FP}\]
  • Precisión (PPV): Confiabilidad del modelo cuando predice la clase positiva. \[Precision = \frac{TP}{TP + FP}\]
  • Medida F (F-measure / F1): Media armónica que combina precisión y sensibilidad en un solo valor. (\(>0.60\) útil) \[F = \frac{2 \times Precision \times Recall}{Precision + Recall}\]
  • Estadístico Kappa (\(\kappa\)): Ajusta la exactitud considerando la posibilidad de aciertos por puro azar. (\(>0.21\) aceptable) \[\kappa = \frac{Pr(a) - Pr(e)}{1 - Pr(e)}\]

Donde \(Pr(a)\) es el acuerdo observado y \(Pr(e)\) es el esperado por azar.

2.2.5 Visualización del Desempeño: Curva ROC y AUC

Cuando un modelo entrega probabilidades, se debe seleccionar un umbral (threshold) para decidir la clase final.

  • Curva ROC: Grafica la Sensibilidad (TPR) frente a la Tasa de Falsos Positivos (\(1 - Especificidad\)) a través de diversos puntos de corte. Permite examinar el compromiso (trade-off) entre detectar positivos y evitar falsos positivos.
  • Área Bajo la Curva (AUC): Resume la curva ROC en un número entre \(0.5\) (sin capacidad de discriminación) y \(1.0\) (clasificador perfecto). Valores entre \(0.8\) y \(0.9\) se consideran excelentes/buenos.

Criterio para elegir el punto de corte para modelos con criterios de probabilidad J-Index (Índice de Youden).

\[J=Sensibilidad+Especificidad-1\]

2.2.6 Regresión Logística (Logit) / Probit

Pertenece a los Modelos Lineales Generalizados (GLM). Predice la probabilidad de pertenencia a una clase binaria utilizando una función de enlace para mapear resultados al rango \((0,1)\).

  • Logit: Utiliza la función sigmoide para transformar el log-odds (logaritmo de la razón de probabilidades) en una probabilidad.

\[\ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 x_1 + ... + \beta_n x_n\]

\[P(Y=1 | X) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 x_1 + ... + \beta_n x_n)}}\]

  • Probit: Utiliza la función de distribución acumulada de la normal.

2.2.7 Naive Bayes

Algoritmo basado en el Teorema de Bayes que considera simultáneamente la información de múltiples atributos para estimar la probabilidad de un resultado.

\[P(Y|X) = \frac{P(X|Y) \cdot P(Y)}{P(X)}\]

  • \(P(Y/X)\) Probabilidad Posterior
  • \(P(Y)\) Probabilidad a Priori
  • \(P(X/Y)\) Verosimilitud
  • \(P(X)\) Marginal

Si se tiene varios predictores (\(X\)) se supone independencia, esto es:

\[P(Y/X) = \frac{P(X_1/Y)*P(X_2/Y)*\ldots *P(X_p/Y)*P(Y)}{P(X)}\]

Su nombre proviene del supuesto “ingenuo” de que todos los predictores son independientes entre sí dados la clase.

2.2.8 k-Nearest Neighbors (kNN)

Es un algoritmo de aprendizaje perezoso (lazy learning) que no construye un modelo abstracto, sino que almacena los datos de entrenamiento. Clasifica nuevas instancias basándose en la clase mayoritaria de sus \(k\) vecinos más cercanos en un espacio multidimensional, calculando distancias (usualmente Euclidianas).

Esta basado en las distancias más cercanas

Fortalezas

  • Sencillo y efectivo
  • Fase de entrenamiento rápido
  • No hace suposiciones sobre la distribución de datos subyacente

Debilidades

  • No produce un modelo
  • Fase de clasificación lenta
  • Requiere una gran cantidad de memoria

Basado en distancias, principalmente la euclideana (cuantitativas, binarias).

\[d(x,y)=\sqrt{\sum (x_i-y_i)^2}\]

  • K representa el número de vecinos cercanos
  • La clase se clasifica según la proporción de clase más alta de los k vecinos más cercanos
  • Se recomienda escalar las covariables a una misma unidad de medida, si existen múltiples unidades de medida.

\[k=||\sqrt{n_{train}}||\]

2.2.9 Árboles de clasificación

Los árboles de clasificación son métodos de aprendizaje supervisado no paramétricos que segmentan el espacio de los predictores en una serie de regiones rectangulares disjuntas mediante reglas lógicas. A diferencia de los modelos lineales, no asumen una forma funcional predefinida (como una línea o curva) entre las variables \(X\) y la respuesta \(Y\).

Su estructura jerárquica emula el proceso de toma de decisiones: cada nodo interno representa una partición basada en una variable, y cada hoja final representa una asignación de clase (ej. “Abandona” o “Se queda”) o una distribución de probabilidad. Utilizan una estrategia de “divide y vencerás” para segmentar los datos en grupos homogéneos mediante reglas lógicas.

La diferencia principal entre los métodos no radica solo en cómo construyen el árbol, sino en cómo combinan múltiples árboles para mejorar la precisión y reducir el error.

Característica CART C5.0 Random Forest
Estructura Árbol Único Ensamble (Boosting) Ensamble (Bagging)
Relación de Árboles Independiente Secuencial: Cada árbol intenta corregir el error del anterior. Paralela: Cientos de árboles independientes votan.
División de Nodos Binaria (siempre 2 ramas) Múltiple (puede tener >2 ramas) Binaria
Criterio de Selección Índice de Gini: Minimiza la probabilidad de clasificación errónea. Entropía / Ganancia de Información: Maximiza la reducción de incertidumbre. Generalmente Gini (pero aplicado a subconjuntos aleatorios).
Manejo del Error Tiende a la Varianza (Overfitting) si es muy profundo. Reduce el Sesgo (Bias) mediante el aprendizaje continuo. Reduce la Varianza mediante el promedio de muchos árboles.

Los criterios de impureza.

  • Criterio C5.0 (Entropía): Busca maximizar la ganancia de información reduciendo el desorden en los datos.

\[Entropy(X) = \sum -p_i \log_2(p_i)\]

  • Criterio Gini: Otra medida común de impureza para realizar divisiones.

\[ Gini(t) = 1 - \sum_{i=1}^{k} p_i^2 \]

El proceso de construcción de un árbol es recursivo y voraz (greedy):

  1. Definición del Nodo Raíz (Primer Nodo): El algoritmo evalúa todas las variables disponibles. Selecciona aquella que logra la mejor separación inicial de las clases según el criterio (Gini o Entropía). Es, por definición, la variable con mayor poder predictivo global.
  2. Particiones Sucesivas: Una vez dividido el primer nodo, los datos resultantes se tratan como “nuevas bases de datos” y el proceso se repite. Un predictor que se usó en el primer nodo puede volver a usarse más abajo con un punto de corte distinto.

2.2.10 Redes Neuronales Artificiales (ANN)

(Ver google trends)

ANN (Artificial Neural Network) es conocido como un método Black Box.

  • El cerebro humano está compuesto por alrededor de 85 mil millones de neuronas
  • Una red capaz de almacenar una enorme cantidad de conocimiento
  • Un gato tiene aproximadamente mil millones de neuronas
  • Un ratón tiene alrededor de 75 millones de neuronas
  • Una cucaracha tiene solo alrededor de un millón de neuronas.

Muchas ANN contienen muchas menos neuronas, generalmente solo varios cientos.

  • Un cerebro de mosca de la fruta con 100,000 neuronas supera el estado actual de ANN.

Una ANN modela la relación entre un conjunto de señales de entrada y una señal de salida utilizando un modelo derivado de nuestra comprensión de cómo un cerebro biológico responde a los estímulos de las entradas sensoriales. Así como un cerebro usa una red de células interconectadas llamadas neuronas para crear un procesador paralelo masivo, la ANN usa una red de neuronas artificiales o nodos para resolver problemas de aprendizaje.

Topología de una red

  • El número de capas (layers)
  • Si se permite que la información en la red viaje hacia atrás
  • El número de nodos dentro de cada capa de la red

La topología determina la complejidad de las tareas que la red puede aprender. Generalmente, las redes más grandes y complejas son capaces de identificar patrones más sutiles y límites de decisión complejos. Sin embargo, la potencia de una red no solo depende del tamaño de la red, sino también de la forma en que se organizan las unidades.

La red neuronal se construye con un grupo de nodos interconectados, que involucra la entrada, los pesos conectados, el elemento de procesamiento y la salida. Las redes neuronales se pueden aplicar a muchas áreas (entrada, salida), como la clasificación, el agrupamiento y la predicción.

Neurona simple
  • Dendritas: \(X\) (input)
  • Axon terminal: \(Y\) (output)
  • Señal de la dendrita: \(w_i\) (peso)
  • Función de activación: \(f\)

\[y(x)=f\left(\sum_{i=1}^n w_i*x_i \right)\]

Características de una red:

  • Función de activación: \(f\)
  • Topología de la red: Arquitectura
  • Algoritmo de entrenamiento: Como se establecen los pesos

Funciones de activación

La función de activación es el mecanismo por el cual la neurona artificial procesa la información y la pasa a través de la red. Así como la neurona artificial está modelada según la versión biológica, también la función de activación está modelada según el diseño de la naturaleza.

En el caso biológico, la función de activación podría imaginarse como un proceso que consiste en sumar la señal de entrada total y determinar si alcanza el umbral de disparo. Si es así, la neurona transmite la señal; de lo contrario, no hace nada. En términos de ANN, esto se conoce como función de activación de umbral, ya que da como resultado una señal de salida solo una vez que se ha alcanzado un umbral de entrada específico.

Activación natural

Aunque la función de activación del umbral es interesante debido a sus paralelismos con la biología, rara vez se usa en redes neuronales artificiales. Liberadas de las limitaciones de la bioquímica, las funciones de activación de ANN se pueden elegir en función de su capacidad para demostrar características matemáticas deseables y modelar relaciones entre datos.

2.2.10.1 Redes neuronales para la clasificación

La red neuronal es una red formada por neuronas artificiales (o nodos). Hay tres tipos de neuronas dentro de la red: neuronas de entrada (\(X\)), neuronas ocultas (variable latentes) y neuronas de salida (\(Y\)). En la red, las neuronas están conectadas; la fuerza de conexión entre las neuronas se llama pesos (\(w\)). Las neuronas de entrada reciben la información de entrada; cuanto mayor sea el valor de entrada, mayor será la activación.

Las ventajas de una red neuronal son: primero, puede detectar relaciones no lineales entre la variable dependiente e independiente. En segundo lugar, uno puede entrenar de manera eficiente grandes conjuntos de datos utilizando la arquitectura paralela. Tercero, es un modelo no paramétrico por lo que se pueden eliminar errores en la estimación de parámetros.

Las principales desventajas de una red neuronal son que a menudo converge al mínimo local en lugar del mínimo global. Además, puede sobreajustarse cuando el proceso de entrenamiento dura demasiado.

Para la clasificación es importante tener presente de manera previa al modelo:

  • Las unidades de medida afectan las funciones de activación, para evitar esto se recomienda estandarizar las variables.
  • En general, el número de nodos ocultos no cuenta con una regla respecto su valor, solo se recomienda no tener demasiados nodos ocultos, ya que esto genera un sobre ajuste. Como recomendación general, se puede tener una cantidad máxima equivalente al número de covariables (X).

Considerados métodos de “caja negra”, imitan la estructura biológica del cerebro con neuronas artificiales interconectadas por pesos (\(w\)).

  • Estructura: Capas de entrada, ocultas (donde ocurre el aprendizaje de patrones complejos) y de salida.
  • Función de Activación: Mecanismo (como la sigmoide) por el cual la neurona procesa la información antes de pasarla a la siguiente capa.
  • Entrenamiento: Utiliza el algoritmo de retropropagación (backpropagation) y el descenso del gradiente para ajustar los pesos y minimizar el error.

\[\theta_{nueva} = \theta_{anterior} - \eta \cdot \frac{\partial E}{\partial \theta}\]

2.2.10.2 Ejemplo del funcionamiento de una red neuronal

  • Dataset con 2 covariables y una y (binaria)
  • La capa oculta tiene 2 neuronas
  • 1 salida con activación sigmoide
  • Pesos aleatorios
  • 2 iteraciones usando el algoritmo backpropagation con descenso del gradiente

\[\sigma(z)=\frac{1}{1+e^{-z}}\] Capa oculta

\[ \begin{aligned} z_{h1} &= w_{11}x_1 + w_{12}x_2 + b_1 \\ a_{h1} &= \sigma(z_{h1}) \\ z_{h2} &= w_{21}x_1 + w_{22}x_2 + b_2 \\ a_{h2} &= \sigma(z_{h2}) \end{aligned} \]

Capa de salida:

\[ \begin{aligned} z_o &= w_{o1}a_{h1} + w_{o2}a_{h2} + b_o \\ \hat{y} &= \sigma(z_o) \end{aligned} \]

SGD es un método de optimización para minimizar la función de error actualizando los pesos con una sola observación a la vez.

\[ \theta^{(t+1)} = \theta^{(t)} - \eta \cdot \nabla_\theta \, \mathcal{L}^{(i)}(\theta) \]

Donde: - \(\theta\): peso o parámetro a ajustar - \(\eta\): tasa de aprendizaje - \(\mathcal{L}^{(i)}\): función de pérdida para la i-ésima observación - \(\nabla_\theta \mathcal{L}^{(i)}\): gradiente (derivada) del error con respecto a ese peso

\[ \begin{aligned} \delta_o &= (\hat{y} - y) \cdot \sigma'(z_o) \\ \delta_{h1} &= \delta_o \cdot w_{o1} \cdot \sigma'(z_{h1}) \\ \delta_{h2} &= \delta_o \cdot w_{o2} \cdot \sigma'(z_{h2}) \end{aligned} \]

\[ \theta_{nueva} = \theta_{anterior} - \eta \cdot \frac{\partial E}{\partial \theta} \]

library(nnet)
# Dataset
x <- data.frame(
  x1 = c(0.00, 0.77),  # edad normalizada
  x2 = c(0.25, 0.67)   # presión normalizada
)
y <- c(0, 1)

#Pesos aleatorios
set.seed(123)
w <- runif(9, min = -0.5, max = 0.5)
names(w) <- c(
  "w11", "w12", "b1",     # pesos para h1
  "w21", "w22", "b2",     # pesos para h2
  "wo1", "wo2", "bo"      # pesos para salida
)
w

## -0.21242248  0.28830514 -0.09102308  0.38301740 
##         w22          b2         wo1         wo2 
##  0.44046728 -0.45444350  0.02810549  0.39241904

#Sigmoide y su derivada
sigmoid <- function(z) 1 / (1 + exp(-z))
sigmoid_deriv <- function(z) sigmoid(z) * (1 - sigmoid(z))

#iteración
iteracion_manual <- function(x1, x2, y_true, w, lr = 0.1) {
  # Propagación
  zh1 <- w["w11"] * x1 + w["w12"] * x2 + w["b1"]
  ah1 <- sigmoid(zh1)
  
  zh2 <- w["w21"] * x1 + w["w22"] * x2 + w["b2"]
  ah2 <- sigmoid(zh2)

  zo <- w["wo1"] * ah1 + w["wo2"] * ah2 + w["bo"]
  y_hat <- sigmoid(zo)

  # Errores
  delta_o <- (y_hat - y_true) * sigmoid_deriv(zo)
  delta_h1 <- delta_o * w["wo1"] * sigmoid_deriv(zh1)
  delta_h2 <- delta_o * w["wo2"] * sigmoid_deriv(zh2)

  # Actualización de pesos salida
  w["wo1"] <- w["wo1"] - lr * delta_o * ah1
  w["wo2"] <- w["wo2"] - lr * delta_o * ah2
  w["bo"]  <- w["bo"]  - lr * delta_o

  # Actualización pesos ocultos
  w["w11"] <- w["w11"] - lr * delta_h1 * x1
  w["w12"] <- w["w12"] - lr * delta_h1 * x2
  w["b1"]  <- w["b1"]  - lr * delta_h1

  w["w21"] <- w["w21"] - lr * delta_h2 * x1
  w["w22"] <- w["w22"] - lr * delta_h2 * x2
  w["b2"]  <- w["b2"]  - lr * delta_h2

  list(y_hat = y_hat, pesos = w)
}
#Primera iteración
w1 <- iteracion_manual(x[1,1], x[1,2], y[1], w)
cat("Predicción 1: ", round(w1$y_hat, 4), "\n")
## Predicción 1:  0.5568
w2 <- iteracion_manual(x[2,1], x[2,2], y[2], w1$pesos)
cat("Predicción 2: ", round(w2$y_hat, 4), "\n")
## Predicción 2:  0.5632
#Segunda iteración
w3 <- iteracion_manual(x[1,1], x[1,2], y[1], w2$pesos)
cat("Predicción 3: ", round(w3$y_hat, 4), "\n")
## Predicción 3:  0.5558
w4 <- iteracion_manual(x[2,1], x[2,2], y[2], w3$pesos)
cat("Predicción 4: ", round(w4$y_hat, 4), "\n")
## Predicción 4:  0.5623
